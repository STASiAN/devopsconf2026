# Заявка для программного комитета DevOpsConf 2026

## Название доклада

**От Kubespray к единой платформе: как Kubernetes, Cluster API и Crossplane связали облако и on-premise в цельную экосистему**

## Формат

Доклад (45 минут)

## Краткое описание

История эволюции инфраструктурной платформы за 5 лет: от ручной установки Kubernetes через Kubespray до полностью автоматизированной системы на базе Cluster API и Crossplane. Практический опыт построения единой платформы, объединяющей облачные и on-premise окружения.

## Целевая аудитория

- DevOps/SRE-инженеры
- Platform Engineers
- Технические лиды инфраструктурных команд
- Архитекторы облачных решений

## Ключевые темы

### 1. Эволюция инфраструктуры
- Путь от Kubespray к современным декларативным инструментам
- Переход на Immutable Infrastructure и Talos Linux
- Отказ от SSH и классических подходов к управлению хостами

### 2. Автоматизация через Kubernetes-native инструменты
- Cluster API для управления кластерами
- Crossplane для упрощения инфраструктуры
- Композитные ресурсы как альтернатива Terraform

### 3. GitOps и Continuous Delivery
- Построение CD-системы (до появления термина "GitOps")
- Декларативное описание через Custom Resources
- Отказ от Helm в пользу версионируемых ресурсов

### 4. Упрощение сложности
- Как снизить порог входа для разработчиков
- Замена HCL/Terraform на YAML и Kubernetes API
- Отказ от Ansible в пользу полной автоматизации

## Практическая ценность

Слушатели узнают:
- **Реальный опыт миграции** с традиционных инструментов (Terraform, Ansible, Helm) на Kubernetes-native решения
- **Архитектурные решения** для объединения облачных и on-premise окружений в единую платформу
- **Метрики эффективности**: как 4 DevOps-инженера за год построили платформу, которую может поддерживать один человек
- **Компромиссы и challenges**: что стоило внедрение GitOps, какие проблемы возникли с Talos Linux, как обеспечить ACL на production

## Уникальность доклада

- История реального production-проекта с начала эры Kubernetes
- Сравнительный анализ подходов: традиционные инструменты vs Kubernetes-native
- Практические кейсы упрощения сложной инфраструктуры
- Честный разговор о проблемах и компромиссах (не только успехи)

## Структура доклада

1. **Вступление** (5 мин)
   - Исторический контекст: Kubernetes в ранние годы
   - Первый опыт с Kubespray и Docker

2. **Эволюция платформы** (15 мин)
   - От Kubespray к Cluster API
   - Попытки с Terraform и IaaS
   - Переход на Immutable Infrastructure (Talos Linux)

3. **Современная архитектура** (15 мин)
   - Crossplane и композитные ресурсы
   - GitOps без Helm
   - Декларативный подход через CRD

4. **Результаты и выводы** (10 мин)
   - Метрики: команда, производительность, поддержка
   - Компромиссы: удобство разработки vs безопасность
   - Уроки и рекомендации

## О спикере

Опыт работы с Kubernetes с момента его раннего adoption. Прошел путь от DevOps-инженера до архитектора платформы. Участвовал в построении инфраструктуры, объединяющей облачные и on-premise решения для enterprise-проектов.

## Демо и интерактив

- Live-демонстрация создания инфраструктуры через Crossplane
- Сравнение сложности: Terraform HCL vs Kubernetes YAML
- Опрос аудитории: "Кто любит Terraform?" (интерактивное начало)
